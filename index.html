Introduction

Welcome to the UI Dev Wiki! The purpose of this guide is to help UX and product designers learn and transition into the field of UI development.

As a UX designer interested in transitioning to UI development, you already have a strong foundation in user experience and design principles. This will be advantageous as you learn the technical aspects of web development. Here's a guide to get you started on your journey:

  1. HTML (Hypertext Markup Language):

    HTML is the backbone of every web page. It defines the structure and content of your web documents. Learn HTML tags, elements, and attributes to create the skeleton of your web pages.

  2. CSS (Cascading Style Sheets):

    CSS is used to style and visually enhance HTML elements. Study selectors, properties, and values to control the layout, typography, colors, and other visual aspects of your web pages.

  3. Responsive Design:

    Learn how to make your websites responsive to different screen sizes and devices. This involves using CSS media queries, flexible grids, and responsive units to ensure your designs adapt seamlessly.

  4. JavaScript:

    JavaScript is a programming language that adds interactivity and dynamic functionality to your web pages. Start with the basics, such as variables, data types, conditionals, loops, and functions. Then progress to more advanced concepts like DOM manipulation, event handling, and AJAX (Asynchronous JavaScript and XML).

  5. UI Frameworks and Libraries:

    Familiarize yourself with popular UI frameworks and libraries like Bootstrap, Foundation, or Material-UI. These frameworks provide pre-designed components, responsive grids, and CSS styling to expedite UI development.

  6. Version Control:

    Git is a widely used version control system that helps you track changes to your code. Learn the basics of Git, including creating repositories, committing changes, branching, merging, and collaborating with others using platforms like GitHub or GitLab.

  7. Web Accessibility:

    Ensure your websites are accessible to all users, regardless of disabilities. Study accessibility guidelines (e.g., WCAG) and learn how to implement accessible features like proper semantic HTML, alternative text for images, keyboard navigation, and ARIA (Accessible Rich Internet Applications) attributes.

  8. Build Tools and Task Runners:

    Familiarize yourself with build tools like npm (Node Package Manager) and task runners like Gulp or Grunt. These tools automate repetitive tasks, such as minifying and concatenating files, optimizing images, and managing dependencies.

  9. UI Design Tools:

    As a UX designer, you may already be familiar with design tools like Adobe XD, Sketch, or Figma. Continue using these tools to create UI mockups and design assets. Export assets in appropriate formats for development, such as SVGs, PNGs, or JPEGs.

  10. Continuous Learning and Practice:

    Web development is an ever-evolving field, so stay updated with the latest trends, technologies, and best practices. Engage in online communities, forums, and blogs. Practice by building real-world projects and experimenting with different UI designs and interactions.

Remember, transitioning from UX design to UI development will require a blend of design skills and technical knowledge. By leveraging your existing UX expertise, you can create user-centered UIs that seamlessly merge aesthetics and functionality. Good luck with your journey into UI development!

Getting Started

If this is your first time learning code then I recommend going through prompts and courses through freecodeacademy.org.

If you are familiar with HTML and CSS and want to practice building websites then this guide might help you.

This webpage was built using HTML and CSS with just a bit of JavaScript. We will focus on HTML and CSS first and then dive into the basics of JavaScript.

Let's practice editing this webpage.

Right click anywhere on the page and select "Inspect." A menu will open revealing the HTML and CSS. In the top section it shows the HTML. Think of this as your layers. Your styles are in the bottom section of the menu.

Your First Website

The first thing you will need to do is install Visual Studio Code or create a Github account. If you have any issues with either of those steps there are tutorials on their sites or you can ask ChatGPT for help.

VS Code is fairly similar to programs like Figma. You've got your "layers" on the left and your "canvas" in the middle. The left menu is your filing structure. You will need to create a dependency (a folder). Name it anything you wish. I named my dependency to create this webpage "dev-wiki." Now you will create two files: index.html and styles.css.
